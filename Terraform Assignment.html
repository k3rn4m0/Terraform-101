STEP 6: Terraform Basics: 

There is a lot of detail to be covered especially with a Cloud infrastructure provisioning/orchestration tool like terraform. Hence, the below points shouldn’t be considered as a deep dive into the basics but just an overview as they’re provided in a terse manner. However, certain links would be provided for orienting the reader to try out some examples:

  • It’s a cloud hybrid tool to build Infrastructure i.e. it can be used with any cloud provided (not only AWS but also GCP & Azure) 
  • A better option compared to cloud formation as it is cloud-agnostic plus it doesn’t put us in vendor lock-ins
  • Uses HCL (Hashicorp Configuration Language) which has a high-level syntax and helps in re-usability of the config files.

  • One of the vital components of the Terraform file is its provider.   • The terraform init command automatically downloads the respective provider which can be verified from the path: ~/AWS/.terraform/plugins/linux_amd64/
